# v0.5 Beta — Release Note

**日期**:2026-07-21
**代號**:Dynamic Layout Engine
**狀態**:Beta(功能收斂,進入穩定驗證階段)

一句話:格局引擎從「固定房間尺寸 + 餘量全給客廳」升級成「房間依基地大小
自適應、客廳過細長自動切出溢位空間」,並建立了可量化的巡檢台(Benchmark)。

---

## 新增功能

### Dynamic Room Area(自適應房間尺寸)
- 新增 [`src/design/room_program.py`](../../src/design/room_program.py):**所有房間尺寸的唯一來源**。
- 每種房間有 `min_area` / `preferred_area` / `max_area`(面積三段)+ `min_width`
  / `min_depth` / `aspect_max`(形狀約束)+ `priority`(優先序)。
- `allocate_areas()` 加權水位法:全員先到 min → 一起漲到 preferred → 漲到
  max → 餘量原封退回當院子(不再無腦塞客廳)。
- `solve_band()` 把面積目標變成「帶進深 + 各格寬度」,形狀(幾×幾)由可用
  寬度決定,不寫死 `room.width`。
- 效果:基地放大時**每一間房間都跟著長大**(改造前只有客廳/天井變大)。

### Living Overflow(客廳溢位)
- 客廳超過 `aspect_max` 時,Layout Engine **重新切分**——切出溢位空間,
  不硬把客廳拉長。
- 溢位幾何在 [`_house_frame`](../../src/design/layout_generator.py) 算出
  (`x_lv`);1F/2F 共用切點,柱網不受影響(輕隔間、不落樑)。

### Program Selector(溢位用途選擇器)
- `select_overflow_program()` 依 **樓層 / 房數 / 是否已有書房 / 是否已有家庭廳
  / 基地大小** 決定溢位當書房 / 家庭廳 / 多功能室 / 儲藏室——**不固定切成
  同一種**。
- 例:1F 北帶已有書房 → 南帶溢位自動改配家庭廳(避免重複)。

### Aspect Constraint 真正參與生成
- `min_width` / `min_depth` / `aspect_max` 從「只有 Benchmark 評分用」升級成
  **真正參與 Layout Generation**:客廳過細長在生成當下就被重新切分,不是
  等 Benchmark 才提出警告。

### Layout Benchmark(格局巡檢台)
- 新增 [`src/design/benchmark.py`](../../src/design/benchmark.py):一次生成 34 組住宅案例,
  量測五面向並產出報告。
- 輸出 `output/benchmark/`:`benchmark.json`(全量測)+ 逐案 DXF/PNG +
  `report.html`(自包含儀表板,縮圖內嵌)。
- 執行:`python -m src.design.benchmark [--limit N] [--no-render]`。

## 修正問題

- **客廳過細長**(Benchmark 首要問題):多樓層非天井客廳長寬比由 **4.95~6.80
  降到 2.13~2.20**。
- **柱網 vs 面積衝突**:`_plan_x_grid` 新增「反過來從隔牆挑軸線」,零孤柱
  by construction;`BAY_RANGE` 上限 4→6,寬房子也對得上隔牆。
- **開口壓柱浮點誤判**:`_blocked` 禁區 +1mm,消除邊界 rounding 誤判。
- **窗躲不開柱**:客廳段雙窗放不下時自動退回單窗(不再讓生成失敗)。
- **家具擋門迴轉**:溢位/緊湊客廳家具改貼南外牆(那道牆沒門),不擋連通口
  與臥室門。

## 已知限制

1. **天井版(深基地)客廳未套 Living Overflow**:P01~P04 的起居室仍是長條
   (長寬比 3.7~4.7)。天井版有另一套骨架(家庭廳繞天井),尚未接溢位。
2. **孝親房版**:1F 南帶是「客餐廳」(併餐),不切溢位,故客廳偏長。
3. **單層路徑**:未套 Living Overflow(單層客廳本來就較方正,長寬比 3.0~3.2,
   非長條,暫不處理)。
4. **寬基地的家庭廳本身偏長**:如 30×13 → 家庭廳 51 m²、長寬比 3.5。客廳本身
   已緊湊達標;家庭廳是次要彈性空間,可長。未來可把大溢位再切多間。
5. **北帶西段書房偏大**:F3 的西段切割會把餐廳餘量全給書房(可達 30+ m²),
   與南帶家庭廳疊在同側,略顯 busy。書房上限未做。
6. **方位約束僅單層**:多樓層路徑未接 `master_corner` / `kitchen_side`。
