# ROADMAP — 通往 AI Building Design Engine

本檔規劃 v0.5 → v1.0 的版本路線。每版列出**目標 / 核心功能 / 不做的事情**。
「不做的事情」跟「要做的事情」一樣重要——它防止範圍蔓延,保證每版能收斂。

> ⚠️ 原始的施工圖階段路線圖(A1~E4,畫圖元素 → 規則式格局 → 自然語言 →
> 網頁化)已完成,見專案根目錄 `ROADMAP.md`(舊)。本檔是**產品版本**路線圖,
> 從 v0.5 Beta(Dynamic Layout Engine)起算。

---

## v0.5 Beta — Dynamic Layout Engine ✅(目前)

### 目標
把格局引擎從「固定尺寸 + 餘量給客廳」升級成「房間自適應 + 客廳過細長自動
重新分配」,並建立可量化的巡檢台。

### 核心功能
- Dynamic Room Area(面積程式 `room_program.py`,min/preferred/max + 優先序)。
- Living Overflow + Program Selector(客廳過細長切溢位,依脈絡選用途)。
- Aspect / min_width / min_depth 真正參與生成。
- Layout Benchmark(34 案,五面向量化,JSON/DXF/PNG/report.html)。
- 既有:多樓層透天 + 集合住宅、天井骨架、自然語言解析、DXF/SVG/PDF、網頁。

### 不做的事情
- ❌ 不做天井版 / 單層的 Living Overflow(留 v0.6)。
- ❌ 不做結構計算(只保證柱網上下對齊,不算配筋/載重)。
- ❌ 不做自動最佳化搜尋(單次生成,seed 決定變體)。

## v0.6 — Layout Quality Engine

### 目標
把 v0.5 Benchmark 揪出的品質缺口補齊——讓「所有路徑、所有基地」都生出
比例合理、動線清楚、家具完整的圖,Benchmark 警告數收斂到接近零。

### 核心功能
- Living Overflow 擴及天井版 / 孝親房版 / 單層路徑。
- 大溢位再細分(多間 overflow),消除殘留長形家庭廳。
- 房間上限治理:北帶西段書房加上限、儲藏/機房不無限長大。
- 動線品質量化指標(走道寬、房到梯距離、門對門淨距)進 Benchmark。
- 家具豐富度提升(空房間補家具、成套規則擴充)。

### 不做的事情
- ❌ 不改變「兩帶 / 三帶」的基本骨架(那是 v0.7 的事)。
- ❌ 不做結構與法規計算。
- ❌ 不引入機器學習。

## v0.7 — Auto Layout Optimization

### 目標
從「單次生成」升級成「在合格解空間裡搜尋最佳解」——給定需求,自動在
seed / 變體 / 骨架選項間搜尋,最大化一個多目標品質分數。

### 核心功能
- 品質評分函式(面積達標率、長寬比、採光、動線、柱網規則性加權)。
- 搜尋器(枚舉/啟發式)在合格方案裡挑最高分,而非只吃第一個可行解。
- 骨架選擇最佳化(兩帶 vs 三帶 vs 其他)由分數決定,不再只看基地深度。
- 可選:BSP 切割表示法(`src/ml/bsp.py` 已有雛形)當搜尋的動作空間。

### 不做的事情
- ❌ 不做「學出來」的佈局(仍是規則 + 搜尋,不是端到端 ML)。
- ❌ 不做結構與法規計算。
- ❌ 不追求全域最佳(可接受啟發式的近似解)。

## v0.8 — Structure Engine

### 目標
從「示意結構(柱網對齊、樓板滿跨)」升級成「真正的結構初步設計」——
構件斷面、載重路徑、基礎。

### 核心功能
- 梁/柱斷面初估(依跨距、載重、材料)。
- 載重路徑檢核(豎向連續、轉換樑辨識)。
- 基礎配置(獨立/連續基腳,依柱位與地耐力參數)。
- 結構圖層產出(配筋示意、斷面標註)。

### 不做的事情
- ❌ 不做精確結構分析(有限元/動力分析)——只做初步設計級。
- ❌ 不取代結構技師簽證(產出是設計輔助,不是合法圖說)。
- ❌ 不做鋼構/SRC 以外的特殊系統(先做 RC)。

## v0.9 — Rule Engine

### 目標
把散在程式各處的「法規/設計規則」抽成**宣告式規則引擎**——規則可讀、可
增刪、可產出合規報告,不再寫死在生成邏輯裡。

### 核心功能
- 規則以資料描述(退縮、建蔽、容積、採光、逃生、無障礙、面積下限…)。
- 合規檢核器:一份設計 → 逐條規則過/不過 + 依據 + 建議。
- 規則庫可依地區/用途切換(不同縣市自治條例)。
- 生成器與檢核器共用同一份規則(生成時遵守、產出時檢核,單一來源)。

### 不做的事情
- ❌ 不宣稱「法規通過保證」(規則庫是輔助,最終以主管機關為準)。
- ❌ 不做全台各縣市完整法規(先做通用 + 一兩個示範地區)。
- ❌ 不做非住宅特殊法規(先住宅)。

## v1.0 — AI Building Design Engine

### 目標
把前面各版整合成「一句話 → 完整可交付的建築設計」的端到端引擎:自然語言
需求 → 自適應格局 + 最佳化 + 結構 + 法規檢核 → 施工圖級 DXF + 報告。

### 核心功能
- 自然語言驅動(現有 Gemini 解析升級成多輪對話設計)。
- 佈局最佳化 + 結構 + 法規 全線打通,一次產出。
- 學習式輔助(以 Benchmark/歷史方案訓練,提出設計建議或初始解)。
- 完整交付物:多樓層平面 + 剖面 + 立面 + 結構 + 數量 + 合規報告。

### 不做的事情
- ❌ 不取代建築師/技師的專業判斷與簽證。
- ❌ 不做施工圖以外的階段(估價細算、標案文件等)。
- ❌ 不做「完全無人監督自動送審」——維持人在迴路。

## 版本原則

- 每版**先收斂再前進**:上一版 Benchmark 全綠、Regression = 0 才開下一版。
- 每版都有**明確不做的事**,避免範圍蔓延。
- 開發流程見 [DEVELOPMENT_GUIDE.md](DEVELOPMENT_GUIDE.md)。
